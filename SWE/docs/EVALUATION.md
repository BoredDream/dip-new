# 方法评估:压缩 / 嵌入 / 攻击与评估

> 从"抗 AI 修图水印"研究视角,对项目三大流程做批判性评价。每条按
> **工程质量**(实现是否扎实正确)与**科学有效性**(能否支撑课题核心命题:
> *深度水印能扛过 AI 再生成,而经典水印不能*)两个维度分别给出。
>
> 一句话总评:**工程成熟度与评估方法学都过硬;但核心命题目前是"架构正确的待证主张",
> 尚未被现有实验证明** —— 原因集中在第 4 节的两个系统性风险。

---

## 1. 压缩:自实现 JPEG 编解码器(`swe/codec/`)

**工程质量:优秀。** 九步全自实现,与 PIL 逐 Q 校准到 ΔPSNR≈0;修了"满块不发 EOB → 解码端整图错位"的真 bug(说明真懂熵编码)。最关键的是 `coeff_hook`:在 **DCT 之后、量化之前**对 Y 通道回调嵌入,是 JPEG 域水印教科书级的正确落点——"在压缩过程中嵌水印"做成了精确可控的注入点,而非压缩后改像素。

**科学有效性:与课题主线关联弱,且有脆性。**

| 问题 | 说明 |
|---|---|
| JPEG 鲁棒 ≠ 抗 AI 修图 | JPEG 是平滑量化型失真;扩散重绘在潜空间**重新合成内容**。codec 真正服务的是经典基线,而经典基线项目自己承认"遇再生成即死"。它是强工程件,但属**侧翼**而非主攻。 |
| 网格对齐是硬约束 | `embed_in_jpeg` 必须裁到 16 的倍数才能保证嵌/取网格一致;任何几何失同步即提取失败。缺同步/模板机制。 |
| 无 JFIF 容器 | 适合测真实 bpp、跑通链路,但无法与现实 .jpg / libjpeg 互通——现实中"图被 AI 工具重存"走标准编解码,这套自有码流接不进。 |
| 仅 Q≤90 校准 | 高质量区与 libjpeg 数值发散。 |

**结论:压缩模块是工程水准的证明,不是"抗 AI"能力的来源。**

## 2. 嵌入:经典 QIM / DCT-QIM-JPEG / 深度潜空间(`swe/watermark/`)

### 2.1 经典族 + DCT-QIM-JPEG —— 正确、诚实,定位是"负对照"

- **DCT-QIM-JPEG(最稳的经典件)**:Δ=3×量化步长,踩中 QIM 抗量化经验法则(Δ>2Q),故"同质量再压缩 acc=1.0"**可信且可证**;重复码+多数表决、密钥选块规范。
- **DFT 修正**:用共轭对称对而非 fftshift —— 很多实现会做错,这里做对了。
- **死穴正是课题靶心**:QIM 对全局幅度/gamma/缩放极敏感,解码端不估计 Δ,任何全局增益漂移即打乱量化格点;而 AI 修图恰恰改全局统计。所以经典族**抗 JPEG、不抗再生成**,作为基线/负对照——定位正确且诚实。

### 2.2 深度潜空间水印 —— "抗 AI"的真正载体,但目前仅到 PoC

- **架构方向对**:冻结 VAE(RoSteALS)+ 小 secret encoder 注入 Δz + 解码器 + **攻击层入环**(VINE 灵魂);残差缩放支持任意分辨率亦忠实移植。
- **一个隐含权衡**:`embed` 在 256 上算残差再**双线性上采样**叠回原图 → 残差天然**低通**。低通残差更易扛"模糊型再生成"(利),但牺牲原分辨率下的容量与高频不可见性(弊)。论文应显式讨论。
- **致命的诚实点**:tiny VAE / CPU 冒烟 / bit_acc≈0.8,README 自己写明"代理攻击下深度 vs 经典并未拉开差距"。即**核心贡献目前是断言,不是证明**;要兑现须 冻结 SD-VAE + GPU + DIV2K 充分训练。

## 3. 攻击与评估(`swe/attacks/` + `swe/eval/`)

**评估纪律:研究级,全项目最亮。** 多图均值±标准差、不可见性(攻击前 PSNR/SSIM/LPIPS)与鲁棒性(攻击后 bit-acc)严格分开、0.5 随机线标注、攻防权衡曲线记录"攻击后画面 SSIM/PSNR"把方法置于保真-鲁棒平面、`records_to_table` 取中/最强档。文档**拒绝预设结论、一切实测自动判定**——科学态度正确。

**但有会动摇结论的弱点(详见第 4 节为重中之重):**

| 弱点 | 影响 |
|---|---|
| 缺干净图 FPR | 深度解码器在**未加水印干净图**上的 bit-acc 未报(应≈0.5),这是 FPR/合理性硬指标。脆弱水印报了误报 0%,深度没有。 |
| 缺上界锚点 | 未跑**真 VINE checkpoint**当上界,无法判断自实现离"应有水平"多远。 |
| LPIPS 静默退化 | 缺 lpips 库时自动用 MSE,若不标注则"感知"指标名不副实。 |
| 几何攻击注定全挂 | 无同步,crop/rotate/scale 是构造性失败,信息量低,不宜直接比较。 |
| 样本量小 | 冒烟仅 4 张图,方差估计弱。 |

## 4. 核心风险:为什么"深度 > 经典"目前未被证明

这两条是决定命题成立与否的关键,务必正视:

1. **主打攻击 `regen_surrogate` 不是再生成攻击。**
   它 = 模糊+重采样+JPEG+噪声。"扩散编辑≈低通"只对一半:Zhao 等的再生成攻击确实利用去噪抹高频,但真扩散 img2img 还会**重新合成新高频细节并产生语义漂移**,模糊代理模拟不出。结果:该代理**对深度水印攻得太轻、对经典攻得偏重**(经典靠中频),因此**没有能力裁决"深度>经典"**。项目已写明——等于承认中心命题当前**未被检验**。

2. **训练攻击 ≈ 测试攻击,存在泄漏。**
   `AttackLayer`(noise/blur/resize/vae)与 `regen_surrogate`(blur/resize/jpeg/noise)高度重叠。**在你测的攻击上训练**会系统性高估鲁棒性、测不出泛化。真正的结果必须留一个**训练中没见过的真 SD img2img**做 held-out。

---

## 5. 验证路线(按性价比排序)

1. **打破训练/测试攻击耦合**:用真 `diffusion_img2img`(GPU)做 **held-out** 评估,训练里绝不放它。**这一条决定整个命题成立与否。**
2. **深度链路换 冻结 SD-VAE + DIV2K/COCO 充分训练**,并跑**真 VINE 当上界**、报**干净图 FPR(≈0.5)**。
3. **几何攻击补同步/模板**,否则这些列不拿来比较。
4. **诚实标注**:报告 LPIPS 是否真用了 lpips;图数提到几十张量级再谈均值±方差。

> 当前定位小结:经典链路是**已坐实的负对照**;深度链路是**架构正确但仅 PoC、且被"代理攻击 + 训练泄漏"双重掩盖**的待证主张。按上表推进即可把"扎实的工程"升级为"能支撑论断的研究"。
