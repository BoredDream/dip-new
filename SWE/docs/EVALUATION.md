# 方法评估:压缩 / 嵌入 / 攻击与评估

> 从"抗 AI 修图水印"研究视角,对项目三大流程做批判性评价。每条按
> **工程质量**(实现是否扎实正确)与**科学有效性**(能否支撑课题核心命题:
> *深度水印能扛过 AI 再生成,而经典水印不能*)两个维度分别给出。
>
> 一句话总评:**工程成熟度与评估方法学都过硬。主线创新(改进版 DwtDct,§2.2)已落地且消融逐档可证;
> 经本地真扩散 img2img(held-out)检验,核心命题"深度抗再生成、经典/改进版均被再生成通杀"在方向上已成立
> (§4)**:深度模型已正经训练(冻结 SD-VAE + GPU + DIV2K 与测试不重叠),20 张 held-out 图上 clean 0.978、
> 真扩散 0.92–0.96;唯画质偏低(PSNR≈23dB),距论文级尚需更大模型/数据/更长训练。

## 1. 压缩:自实现 JPEG 编解码器(`swe/codec/`)

**工程质量:优秀。** 九步全自实现,与 PIL 逐 Q 校准到 ΔPSNR≈0;修了"满块不发 EOB → 解码端整图错位"的真 bug(说明真懂熵编码)。最关键的是 `coeff_hook`:在 **DCT 之后、量化之前**对 Y 通道回调嵌入,是 JPEG 域水印教科书级的正确落点——"在压缩过程中嵌水印"做成了精确可控的注入点,而非压缩后改像素。

**科学有效性:与课题主线关联弱,且有脆性。**

| 问题 | 说明 |
|---|---|
| JPEG 鲁棒 ≠ 抗 AI 修图 | JPEG 是平滑量化型失真;扩散重绘在潜空间**重新合成内容**。codec 真正服务的是经典基线,而经典基线项目自己承认"遇再生成即死"。它是强工程件,但属**侧翼**而非主攻。 |
| 网格对齐是硬约束 | `embed_in_jpeg` 必须裁到 16 的倍数才能保证嵌/取网格一致;任何几何失同步即提取失败。缺同步/模板机制。 |
| 无 JFIF 容器 | 适合测真实 bpp、跑通链路,但无法与现实 .jpg / libjpeg 互通——现实中"图被 AI 工具重存"走标准编解码,这套自有码流接不进。 |
| 仅 Q≤90 校准 | 高质量区与 libjpeg 数值发散。 |

**结论:压缩模块是工程水准的证明,不是"抗 AI"能力的来源。**

## 2. 嵌入:经典 QIM / DCT-QIM-JPEG / 深度潜空间(`swe/watermark/`)

### 2.1 经典族 + DCT-QIM-JPEG —— 正确、诚实,定位是"负对照"

- **DCT-QIM-JPEG(最稳的经典件)**:Δ=3×量化步长,踩中 QIM 抗量化经验法则(Δ>2Q),故"同质量再压缩 acc=1.0"**可信且可证**;重复码+多数表决、密钥选块规范。
- **DFT 修正**:用共轭对称对而非 fftshift —— 很多实现会做错,这里做对了。
- **死穴正是课题靶心**:QIM 对全局幅度/gamma/缩放极敏感,解码端不估计 Δ,任何全局增益漂移即打乱量化格点;而 AI 修图恰恰改全局统计。所以经典族**抗 JPEG、不抗再生成**,作为基线/负对照——定位正确且诚实。

### 2.2 改进版 DwtDct(`ImprovedDwtDct`)—— P2 主创新,逐档可证

- **三处改进可独立开关、可消融**:多低频带(P1 诊断的 #1–7 幸存带)+ 纹理自适应步长 + Watson JND。
  合成步长 `Δ_eff=δ0·texture_gain·jnd_weight·luminance_mask`,**全部只用嵌入后稳定统计量**
  (块均值=DC、非载体高频能量)推算,不自指载体系数 → 盲提取干净准确率严格 = 1.0(单测卡死)。
- **消融实测站得住**(10 张样图全攻击均值):`默认 0.65 → +多频带 0.76 → +纹理 0.82 → +JND 0.81`。
  解读诚实:**鲁棒性增益主要来自"多低频带 + 纹理自适应";JND 档鲁棒性持平**——这是设计使然,
  JND 换的是**可见性**(改进版 PSNR≈49 dB,远高于 SpreadSpec 下界的 31 dB),不是鲁棒性。消融曲线
  应当如此呈现,不应把 JND 包装成"还能更鲁棒"。
- **相对默认 DwtDct 的优势可信**:再生成代理 strength=0.2 下,默认 DwtDct≈0.52(已死)、改进版≈0.82;
  且在**相同画质**下改进版完胜默认(见 `tradeoff.png`)。这是本项目"经典手工方法能逼近到什么程度"的核心证据。
- **局限(须写进报告)**:仍是 QIM 体系 → **抗全局幅度缩放/几何同步弱**;容量随多频带重复下降;
  对真扩散 img2img 的上限仍由深度方法把守(见 §4)。

### 2.3 深度潜空间水印 —— "抗 AI"的真正载体,但目前仅到 PoC

- **架构方向对**:冻结 VAE(RoSteALS)+ 小 secret encoder 注入 Δz + 解码器 + **攻击层入环**(VINE 灵魂);残差缩放支持任意分辨率亦忠实移植。
- **一个隐含权衡**:`embed` 在 256 上算残差再**双线性上采样**叠回原图 → 残差天然**低通**。低通残差更易扛"模糊型再生成"(利),但牺牲原分辨率下的容量与高频不可见性(弊)。论文应显式讨论。
- **致命的诚实点**:tiny VAE / CPU 冒烟 / bit_acc≈0.8,README 自己写明"代理攻击下深度 vs 经典并未拉开差距"。即**核心贡献目前是断言,不是证明**;要兑现须 冻结 SD-VAE + GPU + DIV2K 充分训练。

## 3. 攻击与评估(`swe/attacks/` + `swe/eval/`)

**评估纪律:研究级,全项目最亮。** 多图均值±标准差、不可见性(攻击前 PSNR/SSIM/LPIPS)与鲁棒性(攻击后 bit-acc)严格分开、0.5 随机线标注、攻防权衡曲线记录"攻击后画面 SSIM/PSNR"把方法置于保真-鲁棒平面、`records_to_table` 取中/最强档。文档**拒绝预设结论、一切实测自动判定**——科学态度正确。

**但有会动摇结论的弱点(详见第 4 节为重中之重):**

| 弱点 | 影响 |
|---|---|
| 缺干净图 FPR | 深度解码器在**未加水印干净图**上的 bit-acc 未报(应≈0.5),这是 FPR/合理性硬指标。脆弱水印报了误报 0%,深度没有。 |
| 缺上界锚点 | 未跑**真 VINE checkpoint**当上界,无法判断自实现离"应有水平"多远。 |
| LPIPS 静默退化 | 缺 lpips 库时自动用 MSE,若不标注则"感知"指标名不副实。 |
| 几何攻击注定全挂 | 无同步,crop/rotate/scale 是构造性失败,信息量低,不宜直接比较。 |
| 样本量小 | 当前批次 10 张样图(且仅 2 张唯一样本循环),方差估计弱;定稿需 ~100 张 DIV2K/COCO。 |

## 4. 核心命题的检验:真扩散 held-out 结果(已补)

原先两条风险现已通过**本地真扩散 img2img(SD1.5/GPU)**直接检验,见 `docs/EXPERIMENTS.md §6.1`:

1. **~~主打攻击不是再生成攻击~~ → 已补真扩散。**
   现已用真 SD1.5 img2img(非 CPU 代理)跑了 held-out 攻击。结果**更干净也更有说服力**:strength=0.1 下
   **所有 QIM 类(默认 DwtDct + 改进三档 + DCT-QIM-JPEG)集体跌到 ~0.5(死)**,只有 `Deep-Latent` 幸存(真扩散 0.92–0.96)。
   **这也暴露出 CPU 代理的偏差**:代理只低通、保低频,故改进版(嵌低频)在代理下能赢默认;**真扩散连低频一起重写**,
   改进版的优势随之消失。结论修正为:**改进版的增益针对压缩/模糊/低通退化成立,但真生成式重绘对所有线性 QIM 通杀**
   ——正是"为何仍需深度水印"的硬证据(呼应 Zhao et al., NeurIPS 2024)。

2. **~~训练攻击 ≈ 测试攻击泄漏~~ → 真扩散是真 held-out,且训练集与测试集不重叠。**
   深度模型用冻结 SD-VAE + CNN 解码器 + GPU 在 **DIV2K(100 张)**上训练,**测试用的 22 张 skimage 图它从没见过**,
   攻击层也只含 noise/blur/resize/vae(**没有真扩散**)。它在 20 张 held-out 图上 clean_acc=0.978、真扩散下稳在
   0.92–0.96。即"深度 > 经典(抗再生成)"在**既无训练泄漏、又泛化到新图**的条件下成立。

**仍存的局限**:深度模型画质偏低(PSNR≈23dB / SSIM≈0.72,远不如经典 49–66dB),且训练集仅 100 张、128px;
论文级上界还需 ConvNeXt 解码器 + 256px + 更多数据 + 更长训练 + 真 VINE 对照 + 干净图 FPR(≈0.5)+ ~100 张测试图。
命题方向已验证,画质/精度待提升。

## 5. 验证路线(按性价比排序)

1. **打破训练/测试攻击耦合**:用真 `diffusion_img2img`(GPU)做 **held-out** 评估,训练里绝不放它。**这一条决定整个命题成立与否。**
2. **深度链路换 冻结 SD-VAE + DIV2K/COCO 充分训练**,并跑**真 VINE 当上界**、报**干净图 FPR(≈0.5)**。
3. **几何攻击补同步/模板**,否则这些列不拿来比较。
4. **诚实标注**:报告 LPIPS 是否真用了 lpips;图数提到几十张量级再谈均值±方差。

> 当前定位小结:经典链路是**已坐实的负对照**;深度链路是**架构正确但仅 PoC、且被"代理攻击 + 训练泄漏"双重掩盖**的待证主张。按上表推进即可把"扎实的工程"升级为"能支撑论断的研究"。

## 6. 改进版 DwtDct 的意义(答辩话术)

> 真扩散实验里改进版与默认 DwtDct 一起阵亡,自然要问:**那"改进"的意义在哪?** 下面是诚实、站得住的回答。

**先承认边界**:若**唯一**威胁是扩散全程重绘,则**任何线性 QIM/变换域水印(含改进版)都不是答案**——这条路有天花板,真扩散实验已证明。改进版**不是"抗 AI 再生成的解法"**。但"改进的意义"≠"它能否扛扩散",意义有三层:

**① 改进本身是真实、可证的工程增益——只针对它该管的威胁。**
现实中绝大多数退化不是全程重绘,而是**重压缩 / 平台转码 / 模糊 / 缩放 / 加噪**。在这个最常见威胁类里数据很硬(同等甚至更好的不可见性下,改进版 PSNR≈49dB 仍远超 40dB 隐形线):

| 攻击 | 默认 DwtDct | 改进版 Imp+JND |
|---|---|---|
| 高斯模糊 σ=1.5 | 0.53(死) | **0.88** |
| 缩放 0.5× | 0.49(死) | **0.84** |
| regen 代理 s=0.2 | 0.54(死) | **0.84** |

**② 真正的科学价值,恰恰是那个"负结果"。**
命题(方案 §3.2)从不是"改进版打赢深度",而是**"经典能逼近到哪、为何仍需深度"**。关键在于:我们没拿一个**没调好的弱基线**送死——而是经频带诊断(P1)挑幸存带 + 纹理自适应 + Watson JND,**构造了尽可能强的线性水印**,它在真扩散下**依然集体阵亡**。这把结论从"一个弱基线失败"升级为 **"整个线性变换域水印这一类,对生成式重绘有根本性脆弱"**,堵死"你只是没调好经典方法"的反驳(对上 Zhao et al., NeurIPS 2024)。**一个被严格逼到极限后仍失败的对照,远比随手做的弱对照值钱。**

**③ 它画出了攻防的"分界线"。**
三条曲线合起来回答"谁在什么强度下死":经典默认**一压缩就掉**;改进版把可存活区**往压缩/模糊方向扩一截**(且更隐形);深度把存活区**扩进真再生成区**。"经典抗压缩 → 改进后逼近 → 深度抗再生成"这条带数字的互补结论,就是攻防权衡曲线那张核心图。

**一句话**:改进版的价值,一半是它在**常见威胁(压缩/编辑)下真的更鲁棒且更隐形**,一半是它作为**"最强经典对照"严格论证了深度方法的必要性**。

> 若想让经典线在**再生成**这条战线也有戏,方向不是继续调 QIM,而是方案提到的 **SimuFreeMark/FreqMark 那类"把水印嵌进 VAE 编码后的低频深层特征"**——那已半只脚踏进深度方法。
