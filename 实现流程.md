## 3. ★ 主线 P1:频带存活诊断(决定"水印往哪个频带嵌")【已实现】

这是创新点①、也是 P2 改进的依据:测 img2img(默认纯 CPU 代理)在不同强度下,破坏 DwtDct
嵌入域(Y→DWT-LL→8×8 DCT)的**哪些频带**,据此选"再生成幸存的低频带"。

```powershell

# B. 正式诊断:用你已有的 DIV2K(~100 张,结论才稳)
python scripts\09_diagnose_frequency_survival.py --data-dir data\div2k\DIV2K_train_HR --n-images 100

# C. 有 GPU + diffusers 时,用真扩散 img2img 验证
python scripts\09_diagnose_frequency_survival.py --data-dir data\div2k\DIV2K_train_HR --n-images 100 --use-img2img
```
**产物**:`results\diagnose\freq_survival.png`(频带×强度破坏热图 + 最温和强度的稳定性剖面)+ `freq_survival.csv`。
**控制台**会打印:默认嵌入带 #18(中频)的稳定性排名 + 推荐改嵌的低频带(smoke 跑出 `[1,2,3,4,5,7]`)。

| 参数 | 含义 | 建议 |
|---|---|---|
| `--data-dir` | 诊断用图目录(图片直接放其中,不递归) | 正式结论用 ~100 张 |
| `--n-images` | 参与诊断的图数 | 快测 3~4;定稿 100 |
| `--strengths` | img2img 强度网格(逗号分隔,低段要密) | 默认 `0.05,0.1,0.15,0.2,0.25,0.3` |
| `--use-img2img` | 用真扩散替代 CPU 代理 | 需 GPU + diffusers |

> 可解释性:每根轴都对应课件——DWT-LL=多分辨率(4.7)、8×8 DCT=频域(4.4)、zigzag=频带排序。

## 4. ★ 主线 P2:改进版 DwtDct(诊断频带 + 纹理自适应 + Watson JND)

把经典 DwtDct 骨架做三处**可解释、可独立开关**的改进(支持消融:默认→+多频带→+纹理→+JND):

- **改进①(位置)**:嵌入位置由默认中频 #18 → P1 选出的**低频幸存带**,跨多频带 + 多块重复、多数表决降 BER。
- **改进②(强度·块维度)= 纹理自适应**:QIM 步长按块纹理活动度(块方差)缩放(纹理掩蔽)。
- **改进③(可见性·频带维度)= Watson DCT JND**:用感知模型给每个频带"恰可察觉上限",压住低频可见性。
  Watson 模型已实现于 `swe/watermark/jnd.py`(频率敏感度表 + 亮度掩蔽)。

合成步长 `Δ_eff(块,带) = δ0 · texture_gain(块) · jnd_weight(带,块)`,两维均可从含水印图重算 → 盲提取一致。
> 改进版方法类 `ImprovedDwtDct` 正在 `swe/watermark/` 落地(本步骤为 P2);完成后用法会补到本节,
> 并接入第 5 节的攻击对比与第 8 节的 LPIPS-BitAcc 权衡图。

## 13. Windows 特有问题与排查

| 现象 | 原因 / 解决 |
|---|---|
| 控制台中文乱码 / 偶发 `UnicodeEncodeError` | 先 `$env:PYTHONUTF8=1` 或 `chcp 65001`;脚本已加固,不会因此崩溃 |

| `No module named 'torchvision'` 且用了 `--decoder convnext` | 装 `torchvision`,或改 `--decoder cnn` |
| `--vae sd` / `--use-img2img` 卡在 HuggingFace 下载 | 设 `$env:HF_ENDPOINT="https://hf-mirror.com"`;`HF_HOME` 指到短路径 |
| PowerShell 里 `conda activate` 不生效 | 先 `conda init powershell` 再重开终端;或改用 "Anaconda Prompt" |
| 加载 checkpoint 结构不匹配 | 架构已写入 checkpoint,`06/07` 自动重建;很旧的 checkpoint 会回退 tiny+cnn,请重训 |
| 显存不足 OOM | 调小 `--batch-size`、`--image-size`;或 `--vae tiny` |
| 路径含中文/空格报错 | 路径加引号;尽量放无空格英文路径 |
