# 面向 AIGC 时代的鲁棒图像水印:诊断驱动改进 DwtDct,对标深度潜空间水印
## —— 详细实施方案(课程答辩版)

> 课程:数字图像处理(DIP)大作业。**主线 = 改进经典 DwtDct 水印,使其抗低-strength img2img(AI 再生成)**。
> 在"JPEG + 经典水印 + 测鲁棒性"原型上,做三个**可解释、可落地**的创新:
> ① **诊断驱动的频带选择**(用实验找出"再生成幸存子带");
> ② **改进版 DwtDct**(多低频带 QIM + 纹理自适应强度 + Watson DCT JND 显式可见性约束);
> ③ **经典↔深度攻防对比**(改进 DwtDct 为主角,深度潜空间水印 RoSteALS+VINE 为**上界基线**,课件经典 DCT 水印为**下界基线**),并保留脆弱水印做篡改定位。
>
> 设计原则:**每一步都能映射到课件公式、不引入黑盒**;主体纯 CPU 可跑,深度上界基线可选(需 GPU)。文中算法公式、参数、流程均按可直接实现的标准给出。

---

## 一、项目目标与一句话定位

做一个 **"在低-strength AI 再生成(img2img)下仍能存活的隐形版权水印"**,并讲清"经典手工方法能逼近到什么程度、深度方法强在哪":

- **守课程根基**:自实现 JPEG 九步编解码器与 DWT/DCT(对应课件第 4 章频域 + 4.7 小波);水印嵌在 **DWT-LL 子带的 DCT 频域**;
- **做诊断**:用"频带 × img2img 强度"的破坏诊断实验,**用数据**回答"水印该往哪个频带嵌";
- **做改进**:据诊断结果,把经典 DwtDct 升级为**多低频带 + 纹理自适应 + Watson JND** 的改进版,显著抗低-strength 再生成;
- **做对比**:与课件经典 DCT 水印(下界)、深度潜空间水印(上界)同台对比,量化"经典抗压缩、改进后逼近深度、深度抗再生成"的互补结论;
- **做溯源**:再嵌一个脆弱水印,定位图像被 AI 重绘/篡改的区域。

最终交付:一套系统 + 一份"方法×攻击"的鲁棒性对比报告 + 频带诊断热图 + 攻防权衡曲线 + 篡改定位热图。

## 二、可行性分析

**技术可行性**
- JPEG 九步、DWT(Haar)、DCT、QIM、Watson JND、脆弱水印均为成熟、可手写的经典算法,Python+NumPy 即可实现,**全程可解释、无算法风险**;
- 诊断实验只用已实现的 DWT-DCT 域 + 攻击套件统计系数变化,纯 CPU;
- 深度水印作为**上界基线**采用 RoSteALS 路线(冻结预训练 VAE + 只训小型秘密编/解码器),原文已开源;不作主线,缺 GPU 时用 tiny-VAE 冒烟或直接引用其数值。

**算力可行性**
- **主线(改进 DwtDct + 诊断 + 经典攻击)**:纯 CPU;
- **诊断的真扩散验证 / 深度上界基线**:单张消费级 GPU(RTX 3060/Colab T4)即可;无 GPU 时诊断用 `regeneration_surrogate`(纯 CPU 代理攻击),深度用 tiny-VAE 冒烟。

**数据可行性**
- 公开数据集:DIV2K(高清自然图)、MS-COCO;诊断与评测建议 ~100 张,统一 resize 到 256×256(尺寸取 16 的倍数,保证 DWT/分块对齐)。

**主要风险与应对**(详见第九节)
- **诊断可能指向极低频 → 可见性风险**:最稳频带往往偏低频,直接嵌会偏可见——这正是改进③(Watson JND)的设计动因,把可见性压到"恰不可察觉"。

**结论**:主线纯 CPU、6 周内可完成;深度上界基线按算力可选。

## 三、系统总体架构与建模思路

### 3.1 数据流总览

```
原图 x + 个人信息比特 m(经 ECC 编码)
        │
        ├─[主线 改进DwtDct] Y→DWT-LL→8×8 DCT→【诊断选出的低频带】QIM 嵌入 ─► 含水印图 x_w
        │         (强度 = δ0 · 纹理自适应(块) · Watson-JND(频带,块))
        ├─[下界基线] 课件经典 DCT 加性非盲水印(FF.png)
        ├─[上界基线] 深度潜空间水印(RoSteALS+VINE,嵌 VAE 潜变量)
        └─[脆弱水印] 空域/低频块级校验 ─────────────────────────► 可篡改定位
        │
        ▼   攻击套件(评估用)—— 主战场:低-strength img2img
   经典攻击(加噪/模糊/中值/JPEG再压缩/裁剪/缩放/旋转)
   AI 攻击(VAE往返 / 扩散img2img强度扫描 / 超分 / regen 代理)
        │ 被攻击图 x'
        ▼
   提取器 D → 恢复信息 m' ;  脆弱水印校验 → 篡改热图
        │
   自动评估:PSNR/SSIM/LPIPS(不可见性) + BER/比特准确率/信息恢复率(鲁棒性)
            + IoU/F1(篡改定位) + LPIPS-vs-BitAcc 攻防权衡曲线
```

### 3.2 建模:方法分工(关键)

| 方法 | 嵌入域 | 设计目标 | 在项目中的角色 |
|---|---|---|---|
| 课件经典 DCT 水印(FF.png) | 整图 DCT,加性,**非盲** | 最经典、未抗 AI 再生成 | **下界基线**(我们要超越的起点) |
| 经典 DwtDct(默认中频) | DWT-LL 的 8×8 DCT 中频,QIM,**盲** | 抗 JPEG,但低-strength img2img 即衰减 | **改进前基线** |
| **改进版 DwtDct(本项目主角)** | **DWT-LL 的诊断选出低频带**,QIM + 纹理自适应 + JND | **抗低-strength img2img** | **创新主方法** |
| 深度潜空间水印(RoSteALS+VINE) | VAE 潜变量 z | 抗 AI 再生成(非线性、内容自适应) | **上界基线**(回答"为何不直接用深度") |

核心科学命题与张力:**低-strength img2img 重写高频、部分改中频、基本保留低频/语义;故水印应往"再生成幸存的低频子空间"嵌。但线性水印嵌低频会撞"可见性的墙"——这正是改进③用 Watson JND、以及深度方法(非线性)能赢的根因。**

## 四、模块详细设计:算法原理与具体实现

### 4.1 模块一:JPEG 编解码器(九步)

**作用**:① 课程核心考点(第 4 章 4.4);② 提供 DCT/量化原语;③ "JPEG 再压缩"攻击复用它。

**编码九步(逐步实现)**

1. **色彩空间转换 RGB→YCbCr**(BT.601 全量程)
   - `Y = 0.299R + 0.587G + 0.114B`
   - `Cb = 128 − 0.168736R − 0.331264G + 0.5B`
   - `Cr = 128 + 0.5R − 0.418688G − 0.081312B`
2. **色度下采样 4:2:0**:Cb、Cr 在 2×2 邻域取平均(人眼对色度不敏感)。
3. **分块与电平移位**:8×8 块,每像素减 128。
4. **二维 DCT**:
   ```
   F(u,v) = (1/4)·C(u)·C(v)·ΣΣ f(x,y)·cos[(2x+1)uπ/16]·cos[(2y+1)vπ/16]
   C(0)=1/√2, C(k≠0)=1
   ```
5. **量化**:`F_q(u,v) = round(F(u,v)/Q(u,v))`。标准亮度量化表(Q50)+ 质量缩放 `S=(Q<50)?5000/Q:200−2Q`,`Q'=floor((Q·S+50)/100)`,截断 [1,255]。
6. **Zigzag 扫描**:8×8 → 64 维,低频在前。
7. **DC 用 DPCM、AC 用 RLE**((行程0数, 非零值) + EOB)。
8. **Huffman 熵编码**(标准附录 K 表)。
9. **解码**:逆过程。

> 与标准 JPEG 的一致性,通过脚本 01 的 PSNR/bpp 逐质量对比校准(实测 ΔPSNR≈0.01)。

### 4.2 模块二:经典水印基线(下界 + 改进前基线)

#### 4.2.1 课件经典 DCT 水印(FF.png)= 下界基线
- 流程(课件原理图):原图与**水印图像**各做整图 DCT → **DCT 系数加性组合** `C'=C+α·W` → 反 DCT → 含水印图;检测**需要原始图像**(非盲)。
- 特点:经典、直观,但**非盲、固定带、无感知模型、未抗 AI 再生成** → 作为**我们要超越的下界基线**与动机起点(详见 `优劣势.md`)。

#### 4.2.2 LSB 空域水印(对应第 3 章)
- 嵌入 `c'=(c&0xFE)|b`,提取 `b=c'&1`。容量大、最脆弱 → 下下限基线,亦用于脆弱水印。

#### 4.2.3 经典 DwtDct(盲 QIM)= 改进前基线
**思路**:Y 亮度 → 一级 Haar DWT 取 LL → LL 按 8×8 分块 DCT → 在**中频系数**(默认 (3,2)=zigzag #18)上用 **QIM** 嵌 1 bit/块。**盲提取**(无需原图)。

**QIM 嵌入(对每个承载比特的系数 c、比特 b、步长 Δ)**
```
1. q = round(c/Δ)
2. if (q mod 2) ≠ b:  q = q + sign(c/Δ − q)   # 调到最近的、奇偶=b 的整数
3. c' = q·Δ
提取: q'=round(c'/Δ), b=q' mod 2
```
**鲁棒性设计**:Δ 取得大于目标质量下该位置的 JPEG 量化步长 → 抗再压缩;重复编码 R + 多数表决降 BER;密钥伪随机选块。

> **局限(诊断要回答的)**:默认嵌中频 #18,在低-strength img2img 下表现如何?见模块三诊断。

### 4.3 模块三:频带存活诊断(创新点①,P1)【已实现】

**问题**:在低-strength img2img(或其 CPU 代理)下,DwtDct 嵌入域(Y→DWT-LL→8×8 DCT)的**哪些频带基本不被破坏**?不被破坏的频带 = 应改嵌的位置。

**度量**(对每个 zigzag 频带 k、每个强度 s,在 ~100 张图上平均):
```
abs[k,s] = mean |D_attacked[k] − D_clean[k]|         绝对扰动(决定 QIM 步长下限)
rel[k,s] = abs[k,s] / (mean|D_clean[k]| + ε)         相对扰动(越小越稳→越该嵌)
```

**产物**:频带×强度**破坏热图** + 最温和强度下的稳定性剖面(`scripts/09_diagnose_frequency_survival.py` → `results/diagnose/freq_survival.png`)。

**已得结论(smoke,由 `09` 自动判定打印,非预设数字)**:默认中频 #18 不在最稳区(实测稳定性排名约 **14–16/64**,随抽样张数/图集略有波动),`09` 据此判定"默认位置选得偏、值得改嵌";最稳的是低频带(实测推荐 **#1–7**,`rel<0.1`);扰动随频率、随 strength 单调加重,且 **strength=0.05 就已破坏中高频** → 印证"低 strength 才是真战场"。`09` 还会判定推荐频带最大 zigzag≤12 落在低频区 → 直接线性嵌入偏可见 → P2 需 JND。**正式结论用 ~100 张 COCO(可选真 img2img)定稿。**

> 代码:`swe/eval/diagnose.py`(无黑盒:rgb_to_ycbcr → haar_dwt2 → dct2 → zigzag)。

### 4.4 模块四:改进版 DwtDct(创新点②,P2)

在经典 DwtDct 骨架上做**三处可解释改进**,三者可独立开关 → 支持消融实验:

- **改进①(位置)**:嵌入位置由默认中频 #18 → **诊断选出的低频幸存带集合**(如 #1–7),并跨**多频带 + 多块重复**做多数表决,降 BER。
- **改进②(强度,空间/块维度)= 纹理自适应**:QIM 基础步长按块纹理活动度(块方差/AC 能量)缩放——纹理强的块容忍更大 Δ(藏得住),平坦块用小 Δ(避免块效应)。对应"纹理掩蔽"。
- **改进③(可见性,频率/频带维度)= Watson DCT JND**:用 Watson 1993 感知模型 `JND = t_basic(频带) · 亮度掩蔽(块)` 给每个频带一个**恰可察觉上限**,把低频嵌入的可见性压住(不靠调参,有 HVS 理论支撑)。

合成步长:`Δ_eff(块,带) = δ0 · texture_gain(块) · jnd_weight(带,块)`。两维正交、均可从含水印图重算(块方差/均值/常数表,不自指被嵌系数)→ 保证盲 QIM 提取端步长一致(JND 已实现于 `swe/watermark/jnd.py`;合成步长的改进版类 `ImprovedDwtDct` 为 **P2 待实现**)。

### 4.5 模块五:深度潜空间水印(上界基线,RoSteALS 路线)

**角色变化**:不再是创新主方法,而是**上界基线**——回答答辩必问"既然要抗再生成,为何不直接上深度?"答:深度更强但**是黑盒 + 要 GPU**;我们用信号处理(改进 DwtDct)去逼近它,并量化差距。

**原理**:扩散 img2img 必经 `图→VAE编码→潜空间加噪去噪→VAE解码`。把水印藏进**潜变量 z**(语义表示),与内容纠缠、难被去噪抹除。

**结构(VAE 冻结,只训两个小网络)**
- 冻结预训练 VAE:Φ(x→z)、Ψ(z→图)(取 SD 的 VAE);
- 秘密编码器 E_s(训练):消息 m → 潜空间残差 Δz;
- 秘密解码器 D_s(训练):被攻击图 x' → L 个比特。

**嵌入/提取/训练**
```
z=Φ(x); z_w=z+E_s(m); x_w=Ψ(z_w);  m'=D_s(N(x_w))
L = λ_msg·BCE(m',m) + λ_lpips·LPIPS(x_w,x) + λ_mse·MSE(x_w,x)
```
- **攻击层 N**:{恒等、高斯噪声、模糊、可微 JPEG、**VAE 往返**};VAE 往返是扩散再生成的廉价可微代理(VINE 灵魂);
- 课程式训练:先收敛比特,再加攻击与画质损失;tiny-VAE 供 CPU 冒烟,SD-VAE 供论文级。

### 4.6 模块六:攻击套件(威胁模型:低-strength img2img)

**威胁模型(写进 motivation)**:攻击者要**在保留图像可用性的前提下洗掉水印**;strength 过高=重画另一张图、不构成剽窃。故现实威胁集中在**低-中 strength** 窄带——这决定了实验为何密集采样低段、不测 strength=0.9。

**经典攻击(对照组)**:高斯/椒盐噪声、高斯模糊、中值、**JPEG 再压缩(复用模块一)**、裁剪、缩放、旋转。

**AI 再生成攻击(主战场)**
- **扩散 img2img + 强度扫描**:`strength` 密集覆盖低段 `{0.05,0.1,0.15,0.2,0.25,0.3,0.4}` + 稀疏对照 `{0.6,0.8}`;数学上 `z_t=√(α_t)·z_0+√(1−α_t)·ε`,s=0.2→t≈200,高频基本被重写、低频基本保留;
- **VAE 往返 / 超分往返**;
- **regeneration_surrogate**(纯 CPU 代理:模糊+重采样+JPEG+噪声),无 GPU 时替代真扩散。

### 4.7 模块七:脆弱/半脆弱水印(篡改定位)

- **脆弱(Wong 式)**:每块取 MSB 哈希写入 LSB,任何改动即碎 → 精确定位;
- **半脆弱**:校验信息嵌低频 DCT + 容差阈值 → **容忍 benign JPEG、对语义重绘敏感**,区分"压缩"与"AI 篡改"。

### 4.8 模块八:评估与可视化

**不可见性(攻击前)**:PSNR(>40 dB)、SSIM(>0.95)、LPIPS(越低越好)。

**鲁棒性(攻击后)**:BER / 比特准确率(**0.5=随机=水印死**,图上标虚线);信息恢复率(ECC 后是否完整复原)。

**核心图表(报告必放)**
1. **频带诊断热图**(创新点①的硬核证据);
2. **强度扫描曲线**:比特准确率 vs img2img 强度,多方法同图(FF 下界 / 经典 DwtDct / 改进版 / 深度上界);
3. **攻防权衡曲线**:横轴 **LPIPS(攻击者付出的"内容代价")**、纵轴 BitAcc/BER,标"水印死但图还像"的危险区——**最有分量的结论**;在相同 LPIPS 下,改进版应显著高于经典基线;
4. **消融曲线**:默认 DwtDct → +多频带 → +纹理 → +JND,四档;
5. 篡改定位热图。

**攻击成功阈值(单数字指标)**:定义 `LPIPS<0.1 且 BitAcc<0.7 即攻击成功`,报"每种方法被攻破所需最低 strength"。

**评估纪律**:~100 张测试图报均值±标准差;每 (方法×强度×图) 至少 3 seed 平均;测试集模型未见过;不可见性攻击前算、鲁棒性攻击后算,不混。

### 4.9 模块九:纠错编码(ECC)
个人信息(如 64 位 ID)先用 **Reed-Solomon / BCH** 编码再嵌入,被攻击错几位也能纠正、完整还原 → 得"信息恢复率"。纯 Python 自实现(GF(2^8),Berlekamp-Massey+Chien+Forney)。

## 五、端到端完整工作流程

1. **准备**:整理数据集;自测 JPEG 编解码器(与 PIL 比对 PSNR/bpp)。
2. **诊断(P1)**:跑频带存活诊断,定下"改嵌哪些低频带"。
3. **改进(P2)**:实现 `ImprovedDwtDct`(多低频带 + 纹理自适应 + Watson JND)。
4. **嵌入个人信息**:ID → ECC → 比特串 m → 各方法嵌入。
5. **攻击**:含水印图依次过经典攻击与 AI 攻击(重点低-strength img2img),各强度各 seed。
6. **提取与验证**:解码器自动取 m' 比对(全自动);脆弱水印产生篡改热图。
7. **评估(P4)**:批量算所有指标,生成五张核心图表。
8. **结论**:量化"FF 下界 < 经典 DwtDct < 改进版 → 逼近 → 深度上界",并诚实列改进版局限(容量、抗幅度缩放)。

## 九、风险与应对

1. **诊断指向极低频 → 可见性风险** → 改进③ Watson JND 显式约束;消融对比"有无 JND"的 PSNR/LPIPS,证明 JND 必要。
2. **盲 QIM 自适应步长一致性** → 步长只用嵌入后稳定的块统计(方差/均值/常数表)推算,不自指被嵌系数;单测卡"干净准确率=1.0"。
3. **改进版抗幅度缩放/几何攻击仍弱** → 诚实列入"局限与展望"(可加同步模板/直方图归一化作进阶)。
4. **真扩散推理慢/需 GPU** → 诊断与对比默认用 `regeneration_surrogate` 纯 CPU 代理,GPU 可用时再跑真 img2img 验证。
5. **JPEG 自实现与标准有差** → 标准 Huffman 表 + 与 PIL/libjpeg 比对校准。
6. **脆弱水印对 benign JPEG 误报** → 半脆弱 + 容差阈值。

## 十、参考文献(已核对作者/会议/年份/arXiv/代码)

> 按作用分组,⭐ 为必读。

### 10.1 经典域水印与感知模型(主线来源)
1. ⭐ Xia, Boncelet, Arce. *Wavelet Transform Based Watermark for Digital Images.* Optics Express, 1998. — **DwtDct 的小波域水印理论背景**(主线下界/起点)。
2. ⭐ Watson. *DCT quantization matrices visually optimized for individual images.* SPIE 1993. — **改进③ Watson JND 感知模型的出处**(频率敏感度 + 亮度/对比掩蔽)。
3. Cox, Kilian, Leighton, Shamoon. *Secure Spread Spectrum Watermarking for Multimedia.* IEEE TIP 1997. — **FF.png 那类"整图 DCT 加性扩频"水印的经典出处**(下界基线)。
4. Chen, Wornell. *Quantization Index Modulation (QIM).* IEEE IT 2001. — **盲 QIM 嵌入的理论基础**。
5. `invisible-watermark`(Stable Diffusion 官方水印库,DwtDct 算法)。 — **外部对照基线**(路径 A,可 pip 直接跑)。

### 10.2 深度 / 潜空间水印(上界基线)
6. ⭐ Bui, Agarwal, Yu, Collomosse. *RoSteALS: Robust Steganography using Autoencoder Latent Space.* CVPRW 2023. arXiv:2304.03400. — **深度上界基线蓝本**(冻结 VAE + 小秘密编/解码器)。
7. ⭐ *VINE: Robust Watermarking Using Generative Priors Against Image Editing.* ICLR 2025. — **攻击层入环 + 模糊代理攻击**思路来源(见 `VINE项目核心实现与算法原理分析.md`)。
8. Fernandez et al. *The Stable Signature.* ICCV 2023. — 生成时水印,思路对比。

### 10.3 威胁模型 / 攻击(立意与现成攻击)
9. ⭐ Zhao et al. *Invisible Image Watermarks Are Provably Removable Using Generative AI.* NeurIPS 2024. arXiv:2306.01953. — **核心动机**:再生成攻击证明像素级隐形水印很脆,语义/低频水印更稳。
10. *WAVES / W-Bench:水印鲁棒性基准*。 — 评估协议参考(低-strength img2img、TPR@FPR)。

### 10.4 基础与最新
11. Zhu et al. *HiDDeN.* ECCV 2018. — 学习型水印奠基(编码器-噪声层-解码器)。
12. *SimuFreeMark / FreqMark*:把水印嵌"低频经 VAE 编码后的深度特征" — 印证"低频+语义子空间"方向,与本项目改进思路一致。

### 10.5 建议阅读顺序
**Watson(改进③)→ QIM(嵌入)→ DwtDct/invisible-watermark(基线)→ Zhao 再生成攻击(动机)→ RoSteALS/VINE(上界基线)**。
